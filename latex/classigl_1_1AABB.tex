\doxysubsection{Detailed Description}
\subsubsection*{template$<$typename DerivedV, int DIM$>$\newline
class igl\+::\+AABB$<$ Derived\+V, DIM $>$}
Implementation of semi-\/general purpose axis-\/aligned bounding box hierarchy. 

Determine if collapse the edge {\ttfamily e} would create new intersections.

The mesh (V,Ele) is stored and managed by the caller and each routine here simply takes it as references (it better not change between calls).

It\textquotesingle{}s a little annoying that the Dimension is a template parameter and not picked up at run time from V. This leads to duplicated code for 2d/3d (up to dim).

Insert a (probably a leaf) \doxylink{classigl_1_1AABB}{AABB} {\ttfamily other} into this \doxylink{classigl_1_1AABB}{AABB} tree. 

If {\ttfamily other}\textquotesingle{}s box is contained in this \doxylink{classigl_1_1AABB}{AABB}\textquotesingle{}s box then insert it as a child recursively.

If {\ttfamily other}\textquotesingle{}s box is not contained in this \doxylink{classigl_1_1AABB}{AABB}\textquotesingle{}s box then insert it as a sibling.

It\textquotesingle{}s a very good idea to call either {\ttfamily rotate} (faster, less good) or {\ttfamily rotate\+\_\+lineage} (slower, better) after insertion. Rotating continues to improve the tree\textquotesingle{}s quality so after doing a bunch of insertions you might even consider calling {\ttfamily rotate} on all nodes.

{\ttfamily insert} attempts to minimize total internal surface area. Where as {\ttfamily init} is top-\/down and splits boxes based on the median along the longest dimension. When initializing a tree, {\ttfamily init} seems to result in great trees (small height and small total internal surface area).
